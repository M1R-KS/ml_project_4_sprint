# Multimodal Calories Prediction

Проект для предсказания калорийности блюда по нескольким типам данных: изображению блюда, текстовому описанию ингредиентов и общей массе порции.

Модель объединяет три источника признаков:

- **изображение** блюда — признаки извлекаются предобученной CV-моделью из `timm`;
- **текст ингредиентов** — преобразуется в sentence vector через FastText;
- **масса блюда** — подается в отдельный небольшой энкодер.

Итоговая задача — регрессия: предсказать `total_calories`.

## Состав проекта

```text
.
├── dataset.py          # конфиг, подготовка датафреймов, FastText, transforms, Dataset/DataLoader

├── fasttext_models/    # локальная папка с FastText-моделью
└── results/            # сохранённые веса лучших моделей
```

## Данные

Ожидаемая структура данных:

```text
data/
├── dish.csv
├── ingredients.csv
└── images/
    ├── dish_1561662054/
    │   └── *.png
    ├── dish_1560455030/
    │   └── *.png
    └── ...
```

Файл `dish.csv` должен содержать как минимум следующие поля:

- `dish_id` — идентификатор блюда;
- `ingredients` — список ингредиентов в виде кодов, разделённых `;`;
- `total_calories` — целевая переменная, калорийность блюда;
- `total_mass` — масса блюда;
- `split` — разбиение на `train` и `test`.

Файл `ingredients.csv` используется для расшифровки кодов ингредиентов в текстовые названия. Изображения должны лежать в подпапках, где имя папки соответствует `dish_id`.

## Модель

Архитектура реализована в `utils.py` в классе `BaseMultimodalModel`.

### Текстовая ветка

Текст ингредиентов преобразуется в 300-мерный sentence vector с помощью FastText. Затем вектор проходит через проекцию:

```python
nn.Linear(300, emb_dim)
nn.LayerNorm(emb_dim)
nn.ReLU()
```

### Визуальная ветка

Для обработки изображений используется модель из библиотеки `timm`. По умолчанию:

```python
maxxvitv2_nano_rw_256
```

Модель загружается с предобученными весами и используется как feature extractor:

```python
timm.create_model(
    cfg.image_model_name,
    pretrained=True,
    num_classes=0
)
```

Параметры базовой CV-модели заморожены, обучаются только последующие линейные слои.

### Ветка массы

Масса блюда подается в простой `MassEncoder`, который преобразует одно число в вектор признаков.

### Объединение признаков

Текстовые, визуальные и числовые признаки приводятся к общей размерности `emb_dim`, после чего объединяются поэлементным умножением:

```python
final_emb = text_emb * image_emb * mass_emb
```

После этого общий вектор подается в линейную голову, которая предсказывает калорийность.

## Аугментации изображений

Для train-выборки используются:

- resize до заданного размера;
- горизонтальное отражение;
- `SquareSymmetry`;
- небольшой `CoarseDropout`;
- `ColorJitter`;
- нормализация по ImageNet mean/std;
- перевод изображения в tensor.

Для test-выборки используются только resize, нормализация и перевод в tensor.

## FastText

Проект использует русскую FastText-модель `cc.ru.300.bin.gz`.

Файл модели большой, поэтому его не стоит хранить в GitHub-репозитории. Лучше скачать отдельно и положить, например, в папку:

```text
models/cc.ru.300.bin.gz
```

После этого нужно обновить путь в `MultiModelConfig`:

```python
self.fasttext_model_path = r"models\cc.ru.300.bin.gz"
```

Функция `get_fasttext(cfg)` автоматически проверяет наличие распакованного `.bin`-файла. Если его нет, она распаковывает `.bin.gz` и загружает FastText-векторы через `load_facebook_vectors`.

## Быстрый старт

### 1. Создать конфиг

```python
from dataset import *
from utils import *

cfg = MultiModelConfig()
set_seed(cfg)
```

### 2. Подготовить датафреймы

```python
df_train, df_test = create_dataframes(cfg)
```

### 3. Загрузить FastText

```python
ft_model = get_fasttext(cfg)
```

### 4. Подготовить DataLoader'ы

```python
cfg, train_loader, test_loader = prepare_loaders(
    df_train,
    df_test,
    ft_model,
    model_name='model_1_base'
)
```

### 5. Запустить обучение

```python
model = start_pipeline(cfg, train_loader, test_loader)
```

Во время обучения выводятся train loss, validation loss и время эпохи. Лучшая модель сохраняется в папку `results`.

## Инференс

Для точечного предсказания используется функция `predict_kcal`:

```python
kcal = predict_kcal(
    model=model,
    fasttext_model=ft_model,
    image_path='data/images/dish_id/example.png',
    text='rice;chicken;olive oil;tomatoes',
    mass=250
)

print(kcal)
```

Функция принимает:

- обученную модель;
- FastText-модель;
- путь к изображению;
- строку с ингредиентами;
- массу блюда.

На выходе возвращается предсказанная калорийность.

## Анализ ошибок

Для поиска самых неудачных предсказаний на тестовой выборке используется:

```python
errors = get_top_k_errors(model, test_loader, df_test, k=5)
errors
```

Функция возвращает блюда с максимальной абсолютной ошибкой между истинной и предсказанной калорийностью.

В ноутбуке отмечены основные причины ошибок:

- нестандартный цвет тарелки;
- мелкие калорийные ингредиенты, например орехи;
- блюда, где ингредиенты лежат слоями и плохо различимы на изображении;
- ситуации, когда описание не полностью соответствует изображению;
- случаи, где калорийность может быть связана с невидимыми компонентами, например заправкой.

## Основные параметры конфигурации

Параметры задаются в `MultiModelConfig`:

| Параметр | Значение по умолчанию | Описание |
|---|---:|---|
| `image_model_name` | `maxxvitv2_nano_rw_256` | CV-модель из `timm` |
| `resize` | `256` | Размер изображения |
| `emb_dim` | `128` | Общая размерность эмбеддингов на подачу в голову |
| `batch_size` | `4` | Размер батча |
| `epochs` | `20` | Количество эпох |
| `criterion` | `MAE` | Функция потерь |
| `weight_decay` | `1e-4` | L2-регуляризация для AdamW |
| `seed` | `42` | Фиксация случайности |

Learning rate задается отдельно для разных частей модели:

- `lr_fasttext_proj`;
- `lr_image_proj`;
- `lr_mass_model`;
- `lr_mass_proj`;
- `lr_head`.
